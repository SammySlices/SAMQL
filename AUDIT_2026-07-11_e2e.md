# SamQL E2E deep audit — build 2026-07-11.579

Scope: the two remaining Windows/Edge failures in `load-query-export.spec.ts`
and `nodeflow-stability.spec.ts`, plus the production paths those tests exercise.

## Finding 1 — nested JSON test selected a column the loader intentionally removed

JSON loads default to relational flattening. The E2E loaded a nested fixture
without an explicit `flatten` choice, then queried the original `meta` STRUCT.
That is contradictory: a flattened load may replace the parent nested structure
with relational columns/tables, so the structured-cell viewer could not be
reached reliably. The test now passes `flatten: false`, asserts the returned
columns, and clicks a stable structured-cell hook for `meta` row 1.

Production hardening shipped with the test fix:

- active IDE result grids now pass `result_id` and filters to `DataGrid`, so a
  truncated structured value can fetch its complete value;
- the memoized grid compares that fetch context, preventing a tab/result switch
  from retaining an old result id;
- dict/list cells are serialized as valid compact JSON instead of Python repr;
- structured expand buttons expose column/row identity for accessibility and
  deterministic automation.

## Finding 2 — NodeFlow edit/run/persistence had two timing windows

Node configuration state was mirrored into refs in a passive React effect and
persisted only by a 500 ms timer. Two real user actions could race that design:

1. edit a filter and immediately click Run — the request could use the previous
   render's graph;
2. edit and immediately reload/close — the final timer could be cancelled before
   the write reached localStorage.

The NodeFlow now updates its live graph ref synchronously inside config patches,
runs imperative actions from that live snapshot, persists after a short debounce,
and flushes every tab on `pagehide`, hidden visibility, tab switch, and unmount.
The E2E resets backend state, parses the actual NodeFlow response body, verifies
both `value` and generated `condition`, forces the lifecycle flush, reloads, and
runs again.

## Regression coverage

- DataGrid component test: memoized grid adopts a changed result id.
- NodeFlow persistence unit tests: current-format write and blocked-storage safety.
- Backend test: nested dict/list values become valid JSON strings on the wire.
- Wave 1 contract: explicit non-flattened load, stable viewer hook, lifecycle
  persistence, latest-graph run path, and full-cell wiring are permanent gates.

No product feature was added; this build is stabilization-only.


## Build .580 follow-up

The final structured-cell failure was a production accessibility defect: the
expand button existed but CSS set it to `display:none` until hover. It is now
always rendered, with subdued opacity until hover/focus. The remaining backend
failures were audited individually: Run-all now clears stale cancel state at
entry; cursor fallback avoids a meaningless cancellation probe when no query id
exists; and two source-contract tests were corrected where their frozen
assumptions contradicted newer valid behavior (memoized result-id adoption and
an increased component-test count).
