# Acceleration and format libraries for SamQL distribution builds.
#
# A bare interpreter can still run SamQL on SQLite alone, but every
# ``build.ps1`` / ``build.sh`` distribution build MUST install this entire
# file into the SAME Python that runs PyInstaller, then hard-verify the load
# stack imports. Packages that are missing at packaging time are NOT bundled,
# and recipients of the .exe then hit load/export failures that look like
# "the file works for me but not for my coworker."
#
#   duckdb   -- default analytics engine (CSV / JSON / Parquet / large data)
#   pyarrow  -- Arrow interchange + Parquet acceleration
#   pandas   -- Python node DataFrames (input `df` / assign DataFrame to `out`)
#   orjson   -- REQUIRED fast JSON (NDJSON rewrite + API / result payloads)
#   ijson    -- streaming JSON for large nested loads
#   openpyxl -- .xlsx / .xlsm read + write
#   sqlglot  -- SQL dialect niceties / validation
#   tzdata   -- IANA timezone data for zoneinfo / frozen Windows builds
#   pytz     -- REQUIRED by DuckDB's Python bindings for TIMESTAMPTZ
#               (current_timestamp, NOW(), timestamp-with-tz columns);
#               without it fetch raises "Required module 'pytz' failed to import"
#   pywebview -- native desktop window (SamQL-AppWindow); falls back to browser
#   pyodbc   -- SQL Server connector (Windows builds)
#   pywin32  -- Windows auth / DPAPI helpers used with SQL Server + secrets
#   msal     -- SharePoint sign-in (device code / interactive OAuth)
#   requests + requests-negotiate-sspi -- Windows Integrated Auth for on-prem SP
#
# NOT pip-installable (fetched separately into assistant/runtime/):
#   llama.cpp llama-server CPU build from ggml-org/llama.cpp GitHub releases.
#   Default ``build.ps1`` / ``build.sh`` mode ``-AssistantPack runtime``
#   calls ``tools/assistant_build_pack.py ensure --runtime-only``, which
#   uses ``tools/fetch_assistant_pack.py --skip-model`` (same as
#   ``.\Fetch-SamQL-Assistant.ps1 -SkipModel``) when the runtime is missing.
#   GGUF weights are NOT bundled by default — recipients drop a .gguf into
#   the install-root ``Model/`` folder (or ``assistant/models/``).
