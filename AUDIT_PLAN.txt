SamQL DEEP-AUDIT MASTER PLAN
============================
Written at build 2026-06-25.443. This is the from-scratch program for
auditing ALL of SamQL, subsystem by subsystem. It restarts the process
that produced passes .438-.443 and widens it to everything.

THE METHOD (every pass, no exceptions)
--------------------------------------
1. RECON      map the subsystem's real code paths (grep + read), list
              its invariants and its inputs.
2. GAUNTLET   drive it EXECUTED wherever the sandbox allows: hostile
              inputs, live servers, real locks, real files. Static
              grep only where execution is impossible (DuckDB-native,
              Windows, PyInstaller) -- and say so.
3. FIX        every unearthed bug is fixed in the same pass.
4. ENFORCE    the gauntlet joins tests/run_tests.py permanently, with
              exact statuses/messages/invariants pinned, so the bug
              class cannot return silently.
5. SHIP       one build per pass, full pipeline, changelog = the
              audit report (finds, fixes, clean bills, tradeoffs).

Severity language used in reports: BUG (wrong behavior), GAP
(inconsistent discipline), WART (misleading but harmless), NOTE
(deliberate tradeoff, documented).

THE FULL PROGRAM FROM THE BEGINNING
-----------------------------------
Each pass is one build. Suggested order runs foundation -> data path
-> UI, so later gauntlets can lean on audited lower layers.

PASS 1  engines.py core            locks, recycle, interrupt, pool
        Gauntlet: real-lock contention matrix (reader/writer/cancel),
        recycle-under-load, keepalive lifecycle, throttle restore on
        exception. Extends .438 to every engine method.

PASS 2  loaders + sniffers          csv/json/parquet/xlsx ingestion
        Gauntlet: malformed files (truncated, BOM, mixed encodings,
        zero rows, 1-col, huge headers, dup headers, binary garbage),
        every destination. Assert: clean error or clean load, never a
        traceback; temp files always reclaimed.

PASS 3  flatten + shred             the .422-.431 machinery
        Gauntlet: synthetic nested fixtures covering every planner
        branch (wrapper-only, MAP leaves, scalar arrays, empty lists,
        nulls at every depth, name collisions after sanitize);
        assert plans + generated SQL shapes; executed on sqlite-side
        helpers, static-pinned for DuckDB-native SQL.

PASS 4  query exec + cancel         run_query, dialects, per-run rail
        Gauntlet: cancel at every stage (pre-exec, mid-exec via
        sibling, post-fetch), reuse/read_only flags, dialect
        translation fidelity, statement splitting (the .403 gap
        rules) against pathological SQL (strings with semicolons,
        $$ bodies, comments).

PASS 5  results + filters + sort    _CachedResult and friends
        Gauntlet: filtered_view correctness vs brute force on random
        data (property-style), sort stability, eviction under the 24
        cap while a grid pages, pin/refcount behavior.

PASS 6  jobs + progress + watchdog  server JOBS, opreg, stall scans
        Gauntlet: job lifecycle races (cancel-before-start,
        double-cancel, done-then-cancel), opreg beat/advance math,
        watchdog false-positive window with a deliberately slow op.

PASS 7  filecache + manifest        parquet cache, restart survival
        Gauntlet: CACHE_VERSION bump behavior, partial-write
        recovery (kill mid-copy simulation), manifest add/remove
        roundtrip, orphan sweep.

PASS 8  exports                     csv/xlsx/journal/COPY paths
        Gauntlet: hostile cell content (quotes, newlines, unicode,
        formula-injection prefixes =+-@ for xlsx), empty results,
        wide results; re-read every export and compare cell-exact.

PASS 9  reconcile + pivot           the comparison engines
        Gauntlet: known-answer fixtures (left-only/right-only/
        field-diff counts), null-key semantics, dup-key behavior,
        pivot agg correctness vs SQL brute force.

PASS 10 nodeflow                    every node's SQL generation
        Gauntlet: per-node golden tests -- tiny input table, run the
        node, assert exact output rows; covers all 46 palette types
        so a node regression is caught by name.

PASS 11 server routes (breadth)     every route, not just the .442 set
        Gauntlet: auto-generated fuzz from the ROUTES table itself --
        for each route: empty body, wrong types per known field,
        oversized strings; assert <500 + json always. New routes get
        fuzzed automatically by construction.

PASS 12 FE state machines           App/Notebook/NodeFlow reducers
        Static-heavy (no DOM): invariant sweeps -- every setState
        from async guarded by liveness, every ref map cleaned on
        delete, drag/drop state can't strand (mouseup outside),
        localStorage schema versioned on every key.

PASS 13 packaging + launcher        regen, PS expand, PyInstaller
        Sandbox-static + on-box checklist: regen byte-fidelity
        (already enforced), defang pipeline, launcher log rotation,
        port-in-use behavior, clean-exit signals. Produces a manual
        on-box script for the Windows-only parts.

PASS 14 performance                 the bottleneck ledger
        Gauntlet: timed micro-benchmarks in-suite with generous
        ceilings (paging a 100k result, filter application, catalog
        refresh, journal chain of 20 cells) so a 10x regression
        fails loudly while normal variance passes.

EXIT CRITERIA
-------------
The program is done when every pass above has shipped, every BUG has
an executed regression, and the suite's audit section reads as a
table of enforced invariants -- at which point the audits become
maintenance: re-run free, extended only when a subsystem grows.
